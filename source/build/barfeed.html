barfeed – Bar数据来源¶

class pyalgotrade.barfeed.BaseBarFeed(frequency, maxLen=1024)¶

基类：pyalgotrade.feed.BaseFeed

为 pyalgotrade.bar.Bar 提供数据源的基类。

参数:
  • frequency – K线的频率. 有效定义详见 pyalgotrade.bar.Frequency.
  • maxLen (int.) – pyalgotrade.dataseries.bards.BarDataSeries 类型的值驻留的最大数量。当队列已满，一旦添加新的条目进来，将会在相反位置删除对应数量的条目。

注解

这是一个基类，不应该直接被调用。

getNextBars()¶

class pyalgotrade.barfeed.csvfeed.GenericBarFeed(frequency, timezone=None, maxLen=1024)¶

基类：pyalgotrade.barfeed.csvfeed.BarFeed

一个从CSV文件中加载的BarFeed具有以下格式：

Date Time,Open,High,Low,Close,Volume,Adj Close
2013-01-01 13:59:00,13.51001,13.56,13.51,13.56,273.88014126,13.51001
参数:
  • frequency – bars的频率。详见 pyalgotrade.bar.Frequency.
  • timezone (A pytz timezone.) – 时区设置。默认情况下使用bars自身的时区。详见 pyalgotrade.marketsession.
  • maxLen (int.) – pyalgotrade.dataseries.bards.BarDataSeries 类型的值驻留的最大数量。 存放的最大值。当队列已满，一旦添加新的条目进来，将会在相反位置删除对应数量的条目。

注解

  • CSV文件的第一行 必须 是列名。
  • 复权收盘价 列允许为空。
  • 当多品种同时运行时:
  • 如果加载的所有品种具有相同的时区，则时区参数可以不指定。
  • 如果有任何一个加载的品种具有不同的时区，则时区参数需要设置。

Google Finance¶

class pyalgotrade.barfeed.googlefeed.Feed(frequency=86400, timezone=None, maxLen=1024)¶

基类：pyalgotrade.barfeed.csvfeed.BarFeed

A pyalgotrade.barfeed.csvfeed.BarFeed that loads bars from CSV files downloaded from Google Finance.

参数:
  • frequency – bars的频率. 当前只支持 pyalgotrade.bar.Frequency.DAY 类型。
  • timezone (A pytz timezone.) – 默认情况下使用bars自身的时区. 详见 pyalgotrade.marketsession.
  • maxLen (int.) – pyalgotrade.dataseries.bards.BarDataSeries 类型的值驻留的最大数量。 当队列已满，一旦添加新的条目进来，将会在相反位置删除对应数量的条目。.

注解

Google Finance 的CSV文件中缺少时区信息。

当多品种同时运行时:

  • 如果加载的所有品种具有相同的时区，则时区参数可以不指定。
  • 如果有任何一个加载的品种具有不同的时区，则时区参数需要设置。

Quandl¶

class pyalgotrade.barfeed.quandlfeed.Feed(frequency=86400, timezone=None, maxLen=1024)¶

基类：pyalgotrade.barfeed.csvfeed.GenericBarFeed

A pyalgotrade.barfeed.csvfeed.BarFeed that loads bars from CSV files downloaded from Quandl.

参数:
  • frequency – The frequency of the bars. Only pyalgotrade.bar.Frequency.DAY or pyalgotrade.bar.Frequency.WEEK are supported.
  • timezone (A pytz timezone.) – The default timezone to use to localize bars. Check pyalgotrade.marketsession.
  • maxLen (int.) – The maximum number of values that the pyalgotrade.dataseries.bards.BarDataSeries will hold. Once a bounded length is full, when new items are added, a corresponding number of items are discarded from the opposite end.

注解

When working with multiple instruments:

  • If all the instruments loaded are in the same timezone, then the timezone parameter may not be specified.
  • If any of the instruments loaded are in different timezones, then the timezone parameter must be set.
